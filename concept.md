# 1. PROJECT OVERVIEW

## 1.1 Vision

Build a **premium, enterprise-grade SaaS analytics platform** that goes beyond traditional dashboards by delivering:

- What happened  
- Why it happened  
- What will happen  
- What to do next  

This system transforms raw business data into **decision-ready intelligence**.

---

## 1.2 Core Differentiation

### ❌ Traditional Tools
- “Revenue dropped by 18%”
- “Sales are lower this month”
- “Customer churn increased”

➡️ Surface-level reporting only

---

### ✅ This System

#### 🔍 Level 1 — Insight
“Revenue dropped by 18% primarily due to a 32% decline in returning customers in the electronics category. This correlates with a reduction in repeat purchase campaigns and a 14% drop in discount usage. High-value customers are shifting toward competitors based on pricing patterns.”

---

#### 🧠 Level 2 — Root Cause Analysis
The drop is likely driven by:
- Reduced campaign frequency  
- Higher product pricing vs last quarter  
- Lower engagement from repeat buyers  

---

#### 📈 Level 3 — Prediction
“If this trend continues, projected revenue next month will decline by an additional 12–15%.”

---

#### 💡 Level 4 — Recommendations
To recover growth:
- Reintroduce targeted discount campaigns for repeat buyers  
- Focus on high-margin electronics bundles  
- Launch re-engagement emails for inactive customers  

---

#### 🔮 Level 5 — Scenario Simulation (“What If”)
**User Query:**  
“What if we increase discounts by 10%?”

**System Response:**  
- Revenue increase: +8–12%  
- Customer retention: +15%  
- Margin impact: -3%  
- Net positive growth expected in 2–3 weeks

# 3. PRIMARY PRODUCT FLOW

1. User opens the website  
2. Website appears **premium, elegant, and modern**  
3. Pre-upload sections explain:
   - Product purpose  
   - Pipeline flow  
   - AI usage  
   - Insights generated  
   - Data privacy and deletion  

4. User uploads dataset  
5. Data is sent to AWS backend  
6. Backend processes data:
   - Validation  
   - Transformation  
   - Analysis  
   - Insight generation  

7. Results are displayed in **premium dashboards**  
8. AI assistant enables dataset-based Q&A  
9. User can click **Done / Delete Data**  
10. Full deletion is executed  

---

# 4. DATA DELETION REQUIREMENTS

When user deletes data:

- Remove raw uploaded data  
- Remove processed data  
- Remove dashboard data  
- Remove AI context / embeddings  
- Remove cached outputs  
- Ensure UI reflects deletion
